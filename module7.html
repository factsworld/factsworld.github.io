🎣 Module 7: Captive Portals & Phishing Tricks

Ever connected to public Wi-Fi and got redirected to a login page? That’s a Captive Portal. And guess what? Hackers can create fake ones too—super convincing and ultra sneaky. 😏

📖 What is a Captive Portal?

A captive portal is a web page that users must interact with before gaining full access to the internet. In Evil Twin attacks, we use this to trick users into giving up passwords, emails, or even OTPs!

🔧 Tools to Create Captive Portals

  • wifi-phisher – comes with pre-made templates (Facebook, router login, etc.)
  • CaptiveFlask – customizable fake portals
  • nodogsplash – used in real hotspots, tweak it for phishing

🎭 Real World Example – Bob Tries to Login

You create a fake AP with the SSID "Free_Airport_WiFi". Bob connects and gets redirected to a “Login to get access” page that looks like the airport’s Wi-Fi portal. He enters his email and phone number. And boom 💥—harvested.

📱 Building Your Own Captive Portal

  1. Use wifi-phisher or hostapd + dnsmasq to set up your evil AP
  2. Create a simple HTML login form
  3. Capture credentials to a local file or database
  4. Redirect to “success” page so they don’t get suspicious
// Example: capture.php
file_put_contents("creds.txt", $_POST['email'] . ":" . $_POST['password'] . "\n", FILE_APPEND);

🧠 Tips to Make it Believable

  • Use logos from real companies (but don’t be evil)
  • Match fonts and UI from target sites
  • Delay the redirect a bit to feel more "real"

🚨 Ethical Reminder

This is for **educational use only**. Never trick real users without permission. Stay legal, stay sharp. 🎓

📦 Summary

  • Captive portals are effective for phishing
  • Tools like wifi-phisher simplify the process
  • Always test in lab environments only
