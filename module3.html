Module 3: Setting Up Your Lab

🛠️ Module 3: Building Your Hacking Lab

Ready to go from Wi-Fi noob to packet wizard? First, we need to set up our secret lab — no white coats required, but a hoodie might help the hacker vibe 😎.

💻 Step 1: Choosing Your OS (Kali vs Parrot)

We're going to use a Linux-based OS that comes preloaded with hacking tools. Let’s meet the two contenders:

  • Kali Linux: The OG pentesting distro. Backed by Offensive Security. Clean, focused, tons of tutorials available. We'll use Kali in this course.
  • Parrot Security OS: A little fancier. Lightweight, has anonymous tools built-in. Great for privacy nerds.
💡 Pro tip: You can dual-boot, run it in VirtualBox/VMWare, or live boot with a USB. For best performance, install directly if you can.

📶 Step 2: Choosing a Wi-Fi Adapter

This is crucial. Not all Wi-Fi adapters can go into monitor mode or inject packets (like trying to fly a plane with a bicycle). You need a chipset that supports it.

✅ Recommended Adapters (2025)

  • Cheap: Alfa AWUS036NHA – A legend. Based on Atheros AR9271 chipset. Works flawlessly with Kali.
  • Mid: TP-Link TL-WN722N v1 – Careful: Only v1 works! Later versions use Realtek chipsets (avoid them).
  • Pro: Alfa AWUS036ACH – Dual band, high power. Ideal if you want to flex and capture 5GHz handshakes too.
🧠 Tip: If your adapter doesn’t support monitor mode, you’ll be stuck watching traffic from the sidewalk — not hacking it.

⚙️ Step 3: Installing Required Tools

Kali comes with most tools built-in. But just in case, here's how to update and grab essentials:

sudo apt update && sudo apt upgrade -y
sudo apt install aircrack-ng wireshark net-tools hashcat hcxdumptool hcxtools -y

🧪 Step 4: Test Monitor Mode

Plug in your Wi-Fi adapter and run this:

iwconfig

You should see something like wlan0. Now try:

sudo airmon-ng start wlan0

If you see wlan0mon, congrats — your adapter is a hacker’s friend. If not, time to shop again or check drivers.

🏗️ Step 5: Creating a Safe Test Environment

Never test on networks you don’t own or have permission to test. We suggest:

  • Setting up a second router at home (cheap ones work fine)
  • Giving it a known password to simulate cracking
  • Experimenting on it without fear of breaking anything
🔐 Ethics Reminder: Always hack responsibly. This course is about learning, not law-breaking. We’re teaching skills, not promoting chaos.

✅ Summary

  • We're using Kali Linux for this course (but Parrot is cool too)
  • Choose a good adapter that supports monitor mode + injection
  • Install key tools like aircrack-ng, hcxdumptool, and Wireshark
  • Create your own safe test network to practice everything
